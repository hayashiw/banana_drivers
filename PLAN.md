**SquaredFlux native threshold (2026-04-09):** Ported PedroGil's `threshold` parameter on `SquaredFlux` (in `fluxobjective.py`): `J()=0` and `dJ()` returns zero `B_vjp` once `sq_flux < threshold`. This replaces the earlier `QuadraticPenalty(Jsqf, sqf_target, "max")` wrapping. The threshold is a **noise floor** (~1e-15), not a convergence target ‚Äî 9/10 PedroGil examples use 1e-15 or 1e-16. Setting it to 1e-3 (run 51252548) caused catastrophic coil-coil collapse: Jsqf clipped to 0 early, removing the only descent pressure that prevented banana coils from imploding. `sqf_threshold` is now `1e-15` in config.yaml.

**Ghost $\lambda/\mu$ ratchet diagnosed (2026-04-09):** Run 51251674 showed $\lambda_\text{sqf}=-7.3\text{e}10$ and $\mu_\text{sqf}=3.5\text{e}16$ while `c[sqf]=0` (QuadraticPenalty-wrapped SquaredFlux oscillating above/below `sqf_target=1e-3`). The wrapping created an asymmetric constraint that kept triggering $\mu$ ramp even when "satisfied"; native threshold eliminates this by providing a clean `c=0, \nabla c=0` inactive state.

**`dof_scale` is a coordinate rescaling, NOT a step bound (2026-04-09).** Investigation of SIMSOPT's `augmented_lagrangian.py` confirmed: `dof_scale` maps DOFs to a rescaled space ($y = x / \text{scale}$). Each inner L-BFGS-B iteration is smaller in physical space, but the inner loop runs many iterations (up to `maxfun` function evals) and their displacements accumulate freely. Run 51253798 showed $\|\Delta x\|=0.102$ with `dof_scale=0.005` ‚Äî the raw L-BFGS-B direction had magnitude ~20, and 50 inner iterations accumulated 0.102 physical displacement. The only hard cap on total displacement per outer iteration is `maxfun`.

**ALM preset selector (2026-04-09):** Two presets control the inner-loop philosophy (`stage2_alm.preset`, `BANANA_ALM_PRESET`):
- **`throttled`** (default) ‚Äî `dof_scale=0.1`, `maxfun=100`, `tau=2`, `maxiter_lag=80`. Under-converge inner loop; let outer ramp compensate. Designed for stiff constraint landscapes.
- **`unthrottled`** ‚Äî no `dof_scale`, no `maxfun` cap, `tau=10`, `maxiter_lag=50`. Full inner convergence at each penalty level. Matches PedroGil's examples. Being tested (pending job 51256877 stage 1 + stage 2).

Individual config keys and env vars override preset defaults (resolution: env var > config key > preset).

**Second ALM run failed (job 51228710, 2026-04-08):** Completed 99/100 outer iters in 8:13. Constraint inf-norm 0.706 (tol 1e-6). Final state: length 2.94 m, CC 0.0011 m, curv 41.94 m‚Åª¬π, current drifted to 17.8 kA (cap 16 kA). Lagrange multipliers frozen at `[-8.35e6, 0, 0, -2.17]` from iter 15 onwards; Œº exploded to 1e60 while state was frozen at 9 sig figs across 80 iters. Root cause: **SquaredFlux was the only non-self-clipping term in the constraint list**. Length/CC/curv are wrapped in QuadraticPenalty or are self-clipping threshold penalties (c=0 when below limit), so ALM correctly considers them "satisfied" when inactive. SquaredFlux is a smooth unbounded-below objective that never hits 0, so ALM always considers it active, drives it unboundedly via Œº-ramp, and the resulting descent direction pulls coils into shapes that violate every geometric constraint. The Lagrange-multiplier update rule then fails for the geometric constraints because they're in the violated-but-no-update regime.

**Fix applied (2026-04-08, updated 2026-04-09):** SquaredFlux uses `definition="normalized"` (closes the "shrink the field" loophole) and has a native `threshold` parameter (ported from PedroGil's `simsopt_alm_temp` fork into `fluxobjective.py`): `J()=0` and `dJ()` returns zero `B_vjp` once `sq_flux < threshold`. This gives a clean inactive state (`c=0`, `‚àác=0`) for ALM. Earlier (2026-04-08) this was done via `QuadraticPenalty(Jsqf, sqf_target, "max")` wrapping, but that created a ghost $\lambda/\mu$ ratchet (see 51251125, 51251674). The threshold is `1e-15` ‚Äî a noise floor, not a convergence target (run 51252548 showed that `1e-3` causes catastrophic CC collapse). Banana current pinned at 16 kA (`mode=fixed`).

**Third ALM run ‚Äî wrapping fix confirmed working (job 51236325, 2026-04-08):** 3m42s runtime, hit maxiter_lag. Only curvature violated (42.58 m‚Åª¬π vs limit 40); length 1.113 m and CC 0.066 m both within bounds; current stable at 9.95 kA. Previous blowout of all geometric constraints did NOT repeat ‚Äî the wrapping fix decisively worked. New failure mode is specific and localized: the optimizer makes initial descent (iters 18-20), curvature briefly crosses 40, then state freezes and Œº[curv] ramps to 5.85e49 without further state change. Normalized SquaredFlux reached 1.17e-3 (target 1e-4). Probable root cause: **sqf_target=1e-4 is infeasible at current=10 kA**. The prior successful weighted-mode run (51193701) only reached SQF‚âà1.33e-4 (quadratic-flux units) with current unbounded to 20.89 kA. In ALM mode the current stays at 10 kA (no pressure to increase), so the only way to drive SQF lower is to bend the coils more ‚Äî which drives curvature over the limit. The ALM Lagrange-multiplier machinery gets stuck because any step that reduces curvature also increases the SQF-penalty term. Next iteration options: (a) loosen sqf_target to ~1e-3 (normalized), (b) study whether relaxing curvature or adding current bounds is appropriate.

**Stage 1 phiedge cache bug (2026-04-08).** Warm-start stage 1 was producing wouts with $|B|\sim 12\text{ T}$ (seed wout has ${\sim}1\text{ T}$). Root cause: `Vmec(mpi=mpi)` loads `input.default` at construction, which sets `phiedge=1.0`. The Optimizable `__init__` calls `self.get_dofs()` once and caches `[1.0, 0.0, 1.0]` in `local_full_x`. The driver then overrode `vmec.indata.phiedge = 0.083` from the seed wout ‚Äî but that write bypasses the DOF cache. When `least_squares_mpi_solve` later broadcasts the free-DOF vector via `prob.x = x`, the stale `[1.0, 0.0, 1.0]` silently gets written back through `Vmec.set_dofs`, restoring `phiedge=1.0`. Fix: after the `indata.phiedge = wout_phiedge` override, force-sync the cache with `vmec.local_full_x = np.asarray(vmec.get_dofs())`. Verified by job 51245882: phiedge=0.083, $|B|_\text{max}=1.03\text{ T}$.

**Stage 2 current collapse to 0 kA (job 51246996, 2026-04-08).** First stage 2 ALM run after adding `current` as a 5th constraint (`QuadraticPenalty(|I|, 20\text{ kA}, "max")`) as a warm-start-friendly soft cap. Result: banana current fell from 10 kA to 0 kA on the first inner L-BFGS-B step and froze there for all 80 outer ALM iterations. `grad_norm=0`, `‚ÄñŒîx‚Äñ=0`, no DOF movement after iter 1. Root cause: the warm-start surface from stage 1 gives *lower* normalized SquaredFlux with TF alone ($J_\text{sqf}=1.85\text{e-}3$) than with TF + banana at the initial coil shape ($J_\text{sqf}=2.71\text{e-}3$), because the Fourier-initialized banana coil shape does not yet match what the plasma wants ‚Äî any nonzero current *adds* symmetry-breaking error on top of the near-symmetric TF field. The ALM descent correctly follows the gradient to $I=0$. Once there, the banana B-field is $I \cdot B_\text{unit}(\text{shape})$, so every shape gradient carries a factor of $I$ and vanishes. $\partial J/\partial I$ at $I=0$ is not exactly zero analytically, but `numpy.sign(0)=0` in `CurrentPenaltyWrapper.dJ()` plus the already-zero shape gradients pin the system at a degenerate stationary point. The trivial solution ($I=0$, all constraints 0) is lower-$J$ than anything the optimizer can reach by perturbing shapes.

**Fix applied (2026-04-08):** Stage 2 current handling is now a three-way mode (`current_mode_stage2` in `config.yaml`, `BANANA_CURRENT_MODE_S2` env override):
- **`fixed`** (default) ‚Äî `banana_current` is pinned at `current_fixed_stage2` (default 16 kA, hardware upper bound) via `inner.x *= target/current_now` on the underlying `Current(1)` DOF, then `banana_current.fix_all()` removes it from the free-DOF set. Stage 2 becomes shape-only optimization; singlestage handles joint current + shape refinement against the hardware cap. Matches the "warm-start generator" framing ‚Äî we never needed stage 2 to decide the current.
- **`penalized`** ‚Äî current stays free, `QuadraticPenalty(|I|, current_soft_max_stage2, "max")` added to the ALM constraint list (the configuration of job 51246996). Preserved for experiments but known to collapse to 0 from the current warm-start basin.
- **`free`** ‚Äî current free, no current constraint (legacy pre-51246996 behavior). Also collapses.

`02_stage2_driver.py` only builds `CurrentPenaltyWrapper`/`Jcurr` when `mode == 'penalized'`; in `fixed` mode the DOF is truly removed so the inner optimizer cannot touch it. `constraint_names` is built dynamically so the termination summary and ALM summary JSON remain consistent. `CurrentPenaltyWrapper` was moved to `utils/current_penalty.py` for sharing with `03_singlestage_driver.py`.

## Augmented Lagrangian Method (ALM) ‚Äî Stage 2 port complete

**Motivation.** Stage 2 job 51219280 (2026-04-08) failed because L-BFGS-B's initial descent direction from a fresh `boozersurface.init.json` walked into the `LpCurvCurv` penalty cliff. The `p=4` curvature penalty with threshold 20 m‚Åª¬π creates a sharp nonlinear wall; once the optimizer is on the wall at iter 74, the recommended step blew J up by 14 orders of magnitude (9.6e-4 ‚Üí 5.3e+10), line search could not recover, and the run terminated at machine precision with large gradient. This is starting-point-sensitive and not robust enough for Pareto scans.

**Dead end ‚Äî two-phase warmup (Option B).** Before ALM, a temporary two-phase driver was tried: Phase 1 minimized `SquaredFlux` alone (no penalty walls), Phase 2 enabled the full weighted objective. Phase 1 let the coils grow 11√ó unconstrained (length 0.89 ‚Üí 9.87 m, CC collapse to 4.2 mm, curvature 2197 m‚Åª¬π); Phase 2 could not recover. `SquaredFlux` alone is pathologically unbounded ‚Äî this approach is dead. See job 51221431.

**ALM approach.** Set `f=None` and place **all** objectives ‚Äî including SquaredFlux ‚Äî in the constraint list. ALM treats each as an equality-style residual, ramps per-constraint penalty weights $\mu_i$ in an outer loop, and updates Lagrange multipliers $\lambda_i$ as constraints become satisfied. The effective weight on each constraint's gradient is $(\mu_i c_i - \lambda_i)$, analogous to $w_i$ in a traditional weighted objective. Inner loop is L-BFGS-B on a smooth augmented Lagrangian ‚Äî no penalty cliffs.
